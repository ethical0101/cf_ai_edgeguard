# Prompt Log - cf_ai_edgeguard

This document captures the primary prompts used to build, harden, deploy, and validate the
`cf_ai_edgeguard` project during this implementation cycle.

## 1. Master Build Prompt (Project Foundation)

```text
You are a senior Cloudflare platform engineer and distributed systems architect.

Create a complete production-ready project that fulfills the Cloudflare AI internship assignment.

The project must be an AI-powered application built using Cloudflare platform tools.

Project name:

cf_ai_edgeguard

The project must be fully functional and structured like a real production system.

Never hardcode API keys. Use environment variables.

PROJECT GOAL
Build an AI-powered website security and performance analyzer that runs entirely on Cloudflare infrastructure.

Users should enter a website URL and receive a full analysis report generated by AI.

The system must include:
1. LLM analysis
2. workflow orchestration
3. user input interface
4. memory/state storage

REQUIRED CLOUDFLARE FEATURES
Use Cloudflare Workers, Cloudflare Workflows, Cloudflare KV Storage, Cloudflare Pages,
Workers AI OR external Gemini API.

Implement repository structure, frontend, worker API, workflow steps, scanner module,
AI module, memory module, Wrangler config, docs, setup script, deployment instructions,
and complete production-ready code.
```

## 7. Runtime LLM Prompt (Used by Worker AI Module)

The exact system prompt used in `worker/src/ai.ts`:

```text
You are a cybersecurity and web performance expert.
Analyze the following website headers and response metrics.
Return strict JSON only with this schema:
{
  "security_score": number,
  "performance_score": number,
  "issues": string[],
  "recommendations": string[]
}
Do not include markdown or code fences.
```

## Notes

- All sensitive values must be supplied via environment variables.
- No real API keys should be committed to the repository.
